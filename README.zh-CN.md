# 积分领取 HTML 工具

![Version](https://img.shields.io/badge/version-0.4.0-blue)
![Platform](https://img.shields.io/badge/platform-static%20HTML-orange)

[English](README.md)

一个单文件静态 HTML 工具，用浏览器完成登录、保存登录态，并一键领取可直领积分任务。

> 非官方第三方工具，仅用于个人学习和自动化实验。它与上游服务提供方没有从属、合作、认证或背书关系。

## 为什么做

原 App 里有很多和积分领取无关的功能。这个目录只保留积分领取流程，方便作为 GitHub Pages 静态网页发布，不要把整个 Flutter 项目当成快递箱一起扛上去，太离谱。

## 核心特性

- 手机号、图片验证码、短信验证码登录。
- 使用 `localStorage` 保存 `token`、`uid`、`eid`。
- 支持复制、下载、粘贴导入和文件导入登录信息备份。
- 面向公开使用的风险提示和登录确认。
- 获取短信验证码有 60 秒冷却。
- 可请求浏览器持久存储，尽量延长本机登录态保留时间。
- 获取积分余额和任务列表。
- 通过 `score-send` 串行一键领取。
- 一键领取前会强制刷新任务，并弹窗确认本次任务。
- 用浏览器 JavaScript 移植 MD5 签名。
- 支持每日签到，并尊重服务端 `limits` 次数限制。
- H5 任务不会显示在领取列表里。
- 同一轮一键领取中，每个 `refId` 最多尝试一次。
- `code == -98` 时，当前任务最多重试三次。

## 截图与演示

这是静态页面。可以用 HTTP 服务打开 `index.html`，也可以直接发布到 GitHub Pages。

## 快速开始

本地预览：

```bash
python -m http.server 5173
```

然后打开：

```text
http://127.0.0.1:5173/
```

GitHub Pages 发布：

1. 新建一个 GitHub 仓库。
2. 上传本目录内的文件。
3. 在仓库设置里启用 Pages。
4. 用发布后的地址访问 `index.html`。

## 工程质量

- 不需要包管理器。
- 不需要构建步骤。
- 不包含真实账号、token 或密钥。
- 页面通过浏览器 `fetch` 直接访问上游 API。
- 领取逻辑是串行执行，并带间隔和限流重试。

## 隐私与安全边界

这个工具会把登录信息明文保存在浏览器 `localStorage`。这很方便，但不适合高安全场景。如果别人能访问你的浏览器配置文件，就可能看到登录态。

下载出来的登录备份是包含账号 token 的明文 JSON 文件。只建议保存在你自己的私有设备上，不要上传到 GitHub、网盘笔记、工单、聊天窗口，也别截图发人。这个坑不用踩，踩了就不是技术问题，是保管问题。

登录态只属于同一个浏览器配置和同一个发布域名。保持固定的 GitHub Pages 域名，并让浏览器允许持久存储，可以尽量延长本机自动恢复时间。但它挡不住手动清理站点数据、隐私模式清理、更换浏览器、更换域名，或者上游 token 服务端过期。

不要上传真实 token、包含 token 的截图、复制出来的登录 JSON。

短信登录仍然需要手动完成。本项目不绕过图片验证码或短信验证码。

- 登录态过期时给出更清楚的错误提示。

## 故障排查

- 如果任务加载失败，请用 `http://` 服务或 GitHub Pages 打开，不要用 `file://`。
- 如果登录态丢了，导入之前下载的登录备份，或者重新短信登录。
- 如果领取连续失败，先刷新任务，再判断是不是上游 token 已过期。
